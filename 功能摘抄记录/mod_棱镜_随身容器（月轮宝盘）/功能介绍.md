# 随身容器（月轮宝盘）

## 概述

棱镜 Mod 的月轮宝盘（`revolvedmoonlight`）是一个**在物品栏中打开、不需要动作维持、可直接收起**的随身容器。与原版熊罐（`beargerfur_sack`）走同一条 RPC 通道，但关键机制不同——**不设 `portablestorage` 标签**，因此避开了标签带来的持久动画和动作锁定。

## 核心机制

### 去掉 `portablestorage` 标签是设计的核心

`portablestorage` 标签在原版中不止标记"这是一个随身容器"，它还在 **stategraph 层** 决定了角色动画路由：

```lua
-- SGwilson.lua:1138-1151 — RUMMAGE 动作的 ActionHandler
ActionHandler(ACTIONS.RUMMAGE, function(inst, action)
    if action.invobject then
        if action.invobject:HasTag("portablestorage") then  -- ← 路由分叉点
            return "start_pocket_rummage"                    -- 持久 build 动画
        end
    end
    return "doshortaction"                                   -- 简短弯腰动画
end),
```

带标签 → `start_pocket_rummage` 状态（`build_pre→build_loop` 反复弯腰翻找）→ 锁定动作、移动自动关闭。
不带标签 → `doshortaction`（标准弯腰取放动画）→ 不锁定动作、打开后角色自由移动。

月轮宝盘选择了后者。

### OnUpdate 的联动

`container.lua:1059-1083` 的 `OnUpdate` 每帧检查 portablestorage 标签：

```lua
if self.inst:HasTag("portablestorage") and
   not (opener.sg and opener.sg:HasStateTag("keep_pocket_rummage")) then
    self:Close(opener)
end
```

去掉标签后，此分支永远不命中，关闭只靠"走远超过距离"或手动 X。
与棱镜月轮宝盘无关的 `droponopen`、`skipautoclose` 等参数同样为 nil。

### 棱镜的额外机制（非必需）

月轮宝盘还实现了 AutoOpen——物品栏关闭时记下开关状态、物品栏重新打开时自动恢复。这个逻辑与本笔记无关，暂且不用参考。

### Prefab 关键代码 — `moonlight_legion.lua:825-877`

```lua
fns.MakeRevolved = function(sets)
    table.insert(prefs, Prefab(sets.name, function()
        local inst = CreateEntity()
        -- ...基础实体组件...
        MakeInventoryPhysics(inst)

        -- 不设 portablestorage 标签
        -- inst:AddTag("meteor_protection")
        -- inst:AddTag("nosteal")

        inst:AddComponent("inventoryitem")
        inst.components.inventoryitem.imagename = sets.name
        inst.components.inventoryitem.atlasname = "images/inventoryimages/"..sets.name..".xml"

        inst:AddComponent("container")
        inst.components.container:WidgetSetup(sets.name)
        inst.components.container.onopenfn = pas.OnOpen_revolved
        inst.components.container.onclosefn = pas.OnClose_revolved
        inst.components.container.skipclosesnd = true
        inst.components.container.skipopensnd = true
        -- 不设 droponopen, 不覆盖 OnUpdate
    end))
end
```

### onopen/onclose 回调 — `moonlight_legion.lua:569-591`

```lua
pas.OnOpen_revolved = function(inst, data)
    inst.AnimState:PlayAnimation("open")
    inst.AnimState:PushAnimation("opened", true)
    inst.SoundEmitter:PlaySound("dontstarve/cave/mushtree_tall_spore_land", nil, 0.6)
end

pas.OnClose_revolved = function(inst, doer)
    inst.AnimState:PlayAnimation("close")
    inst.AnimState:PushAnimation("closed")
    inst.SoundEmitter:KillSound("idlesound1")
    inst.SoundEmitter:KillSound("idlesound2")
    inst.SoundEmitter:PlaySound("dontstarve/cave/mushtree_tall_spore_land", nil, 0.6)
end
```

注意：`onopen`/`onclose` 只做**实体自身的动画和音效**，不操作玩家的 stategraph。这是与熊罐最明显的区别之一。

## 与携带式衣柜的关系

我们在携带式衣柜的开发中最初参照了熊罐（`portablestorage` + `droponopen` + `OnUpdate` 覆盖），踩了大量坑后参照月轮宝盘改为不设标签，问题全部自然消失。

详见 [原版类似功能.md](../mod_棱镜_随身容器（月轮宝盘）/原版类似功能.md) 中熊罐 vs 月轮宝盘的全面对比。
