# 测试方法

## 测试 Mod

本功能摘抄来自对「便携衣柜」Mod 开发过程的实测反馈。便携衣柜实现了一个 2行×3列 的随身容器，早期参照熊罐（`portablestorage` 标签方案），后期参照月轮宝盘（无标签方案）。

## 测试环境

| 项目 | 值 |
|------|----|
| 游戏版本 | DST（2026-06） |
| 启用的 Mod 列表 | 便携衣柜（测试目标） |
| 世界设置 | 地面森林世界，无洞穴 |
| 测试角色 | 温蒂（Wendy） |

## 实测反馈

以下按问题发现时间排列。

### 测试 1：`portablestorage` 标签导致移动自动关容器

**操作**：右键物品打开容器 → 移动角色。

**现象**：角色一动容器就关闭。日志显示 `container.lua:1075` 的 OnUpdate 自动触发了 Close。

**对照原版**：熊罐、钓具箱都是这样的行为——这是 portablestorage 标签的**设计意图**（打开需要蹲下不动，移动 = 关闭）。

**结论**：需要移动不关容器，不能加 portablestorage 标签。参照月轮宝盘。

### 测试 2：`skipautoclose = true` 导致扔地上打开走远也不关

**操作**：物品版扔地上 → 打开容器 → 走远。

**现象**：容器永远不关闭。

**原因**：`skipautoclose` 跳过 OnUpdate 的 StartUpdatingComponent，所有自动关闭逻辑都停了。

**结论**：草率方案，一地问题。参照月轮宝盘不需要设这个。

### 测试 3：`keep_pocket_rummage_mem_onexit` + `DoTaskInTime(0)` — 部分解决但动画有缺陷

**操作**：`onopen` 中同时设 `keep_pocket_rummage_mem_onexit = true`（防 OnUpdate 自动关）和 `DoTaskInTime(0) → GoToState("idle")`（跳过持久 build 动画）。

**现象**：容器不会随移动关闭，但画面一闪而过 `build_pre` 动作后才回到 idle。关闭动画变瞬间 cut 而非完整弯腰收回。

**原因**：
- `keep_pocket_rummage_mem_onexit` 让 stategraph 退出时不触发 Close，但 `DoTaskInTime(0)` 在 1 帧内推回 idle，`start_pocket_rummage` 的 onupdate 失去对容器状态的检测，导致关闭动画（`stop_pocket_rummage` 的 `build_pst`）不会被自然触发
- `DoTaskInTime(0)` 的最小延迟是 1 帧，`build_pre` 的残留不可避免

**结论**：参照月轮宝盘不需要这些操作。

### 测试 4：手动 `GoToState("stop_pocket_rummage")` 修复关闭动画 — 不完美

**操作**：在测试 3 基础上，`onclose` 中手动 `doer.sg:GoToState("stop_pocket_rummage")` 播放完整 `build_pst` 关闭动画。

**现象**：关闭动画正常播放，但角色从 idle → `stop_pocket_rummage` → `build_pst` → idle 的切换有一个微妙延迟。

**原因**：正常流程中 `start_pocket_rummage` 的 onupdate 检测到 IsOpenedBy 变 false 后自动推入 `stop_pocket_rummage`。我们已 GoToState("idle") 跳过 start_pocket_rummage 后这个链路断了，所以需要手动推。

**结论**：虽然能工作，但代码复杂且与 stategraph 的耦合极重。参照月轮宝盘完全不需要这些操作。

### 测试 5（最终方案）：去掉 `portablestorage` 标签 — 一切正常

**操作**：去掉 Prefab 中的 `inst:AddTag("portablestorage")`、`droponopen`、`OnUpdate` 覆盖、`onopen`/`onclose` 中的 sg 操作。

**现象**：
- 打开：角色播放 `doshortaction`（标准弯腰取放动画），短而自然，不锁定动作
- 关闭：角色播放 `doshortaction`，完整的弯腰放物动画
- 移动：容器保持打开
- 扔地上：走远自动关（`OnUpdate` 原版距离检测正常工作）
- 拾取：左键正常拾取
- 部署：右键拖出部署正常

**结论**：框架默认的 `doshortaction` 路由完美匹配"快开快关"的交互模式。棱镜月轮宝盘的选择正确。

### 关键对比：portablestorage 标签的作用

| 层级 | 有标签 | 无标签 |
|------|--------|--------|
| stategraph 路由 | `start_pocket_rummage`（持久弯腰动画） | `doshortaction`（简短弯腰动画） |
| 动作锁定 | ✅ `busy` 标签锁定 | ❌ 短暂后释放 |
| OnUpdate 行为 | 移动自动关 | 距离检测关 |
| 掉落保护 | 需 `droponopen=true` | 自然不掉落 |
| 代码简洁度 | 需要大量覆盖/修补 | 零额外代码 |
