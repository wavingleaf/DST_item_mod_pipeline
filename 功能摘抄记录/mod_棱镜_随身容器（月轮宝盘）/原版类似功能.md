# 原版类似功能：熊罐（beargerfur_sack）

## 概述

原版 DST 的熊罐（`beargerfur_sack`）是一个带 `portablestorage` 标签的随身容器。在物品栏中右键打开时，利用 `droponopen=true` + `portablestorage` 标签的组合跳过落地，然后通过 stategraph 的 `start_pocket_rummage` 状态播放持久的弯腰翻找动作。

## 与月轮宝盘的关键差异

| 维度 | 原版熊罐 | 棱镜月轮宝盘 |
|------|----------|-------------|
| `portablestorage` 标签 | ✅ 有 | ❌ 无 |
| `droponopen` | ✅ true | ❌ 不设 |
| 开启动画 | `start_pocket_rummage`：`build_pre→build_loop` 持久弯腰 | `doshortaction`：简短弯腰取放 |
| 关闭动画 | OnUpdate 强制打断时无完整动画 | `doshortaction` 完整弯腰放物 |
| 动作锁定 | 是（`busy` 标签） | 否 |
| 移动时 | 自动关闭容器 | 容器保持打开 |
| 扔地上走远 | 距离检测自动关（同） | 距离检测自动关（同） |
| `skipclosesnd`/`skipopensnd` | ✅ 用自定义音效 | ✅ 用自定义音效 |
| 容器 type | `chest` | `box_legion`（棱镜自定类型，UI 拖拽用） |

## 核心机制链路

### 1. 打开端口（actions.lua RUMMAGE.fn）

```lua
-- actions.lua:1065-1093 — 打开容器时不落地
local owner = targ.components.inventoryitem:GetGrandOwner()
if owner and
    (targ.components.container.droponopen) and
    not (owner:HasTag("player") and targ:HasTag("portablestorage"))
then
    -- 落地逻辑 …
end
```

`droponopen=true` + `portablestorage` 标签 + owner 是玩家 → **跳过 DropItem** → 容器留在物品栏中打开。

### 2. stategraph 动画路由（SGwilson.lua:1138-1151）

```lua
ActionHandler(ACTIONS.RUMMAGE, function(inst, action)
    if action.invobject then
        if action.invobject:HasTag("portablestorage") then
            -- 路由到 start_pocket_rummage（持久 build_pre→build_loop 动画）
            return action.invobject.components.container:IsOpenedBy(inst)
                and "stop_pocket_rummage"
                or "start_pocket_rummage"
        end
    end
    return "doshortaction"
end),
```

`portablestorage` 标签 → `start_pocket_rummage` 状态：
- 播放 `build_pre`（弯腰）→ `build_loop`（持续翻找）
- `tags = { "doing", "busy", "nodangle", "keep_pocket_rummage" }`
- timeline `FrameEvent(7, ...)` — 第 7 帧移除 `busy` 标签（animation done），之后角色可以移动，但 `keep_pocket_rummage` 标签保持

### 3. 自动关闭（container.lua:1059-1083 OnUpdate）

```lua
if self.inst:HasTag("portablestorage") and
   not (opener.sg and opener.sg:HasStateTag("keep_pocket_rummage")) then
    self:Close(opener)
    opener:PushEventImmediate("ms_closeportablestorage", { item = self.inst })
end
```

`portablestorage` 标签 + 玩家不在 `keep_pocket_rummage` 状态 → **自动关**。
换句话说：只要角色移动/被打断导致 sg 离开 `keep_pocket_rummage` 状态，容器秒关。

### 5. Prefab 关键代码（beargerfur_sack.lua:152-208）

```lua
local function fn()
    local inst = CreateEntity()
    -- …基础实体组件…

    inst:AddTag("portablestorage")  -- ← 核心标签

    inst:AddComponent("container")
    inst.components.container:WidgetSetup("beargerfur_sack")
    inst.components.container.onopenfn = OnOpen
    inst.components.container.onclosefn = OnClose
    inst.components.container.skipclosesnd = true
    inst.components.container.skipopensnd = true
    inst.components.container.droponopen = true  -- ← 不落地

    inst:AddComponent("preserver")  -- 冷藏功能（熊罐特有）
    inst:AddComponent("inventoryitem")
end
```

### 6. onopen/onclose 回调

```lua
local function OnOpen(inst)
    inst.AnimState:PlayAnimation("open")
    inst.components.inventoryitem:ChangeImageName(skin_name .. "_open")
    -- …音效和冰冻特效启动…
end

local function OnClose(inst)
    inst.AnimState:PlayAnimation("close")
    inst.AnimState:PushAnimation("closed", false)
    inst.components.inventoryitem:ChangeImageName(skin_name)
    -- …音效和冰冻特效关闭…
end
```

注意：熊罐的 onopen/onclose 只处理实体自身的动画和特效，不操作玩家 stategraph（玩家动画由 stategraph 自己根据 ActionHandler 路由处理）。

## 适用场景判断

| 场景 | 推荐方案 |
|------|----------|
| 角色需要持续蹲守的容器（冰箱背包、钓具箱） | 熊罐（`portablestorage` 标签） |
| 快开快关的容器（装备收纳、随身物品箱） | 月轮宝盘（无标签） |
| 打开后需要走动（边走边看） | 只能选月轮宝盘方案 |
