# 范围边界进出识别

## 概述

在 DST 中，让"玩家（或某个实体）进入某物体一定距离内时触发某些功能"是一个常见需求。游戏本体提供了**两条实现路径**：

| 路径 | 核心 API | 适用场景 |
|------|----------|----------|
| **包装方案** | `playerprox` 组件 | 玩家进出范围即触发回调，即插即用，80% 的 Mod 需求首选 |
| **DIY 方案** | `FindClosestPlayerToInst()` + `inst:IsNear()` | 需要自定义检测逻辑/频率/目标类型（非玩家实体） |

两条路径可共存——同一个实体可以挂 playerprox 做唤醒/休眠，同时用自己的定时器做更精细的距离判断。

## 包装方案：playerprox 组件

### 最简用法

```lua
-- 在 Prefab 的 fn() 中（服务端部分）：
inst:AddComponent("playerprox")                      -- 挂载组件
inst.components.playerprox:SetDist(8, 10)            -- near=8（进入距离）, far=10（离开距离）
inst.components.playerprox:SetOnPlayerNear(function(inst, player)
    -- 任一玩家进入 8 距离内时触发
    print("玩家 " .. player:GetDisplayName() .. " 靠近了！")
end)
inst.components.playerprox:SetOnPlayerFar(function(inst)
    -- 所有玩家都离开 10 距离外时触发
    print("所有玩家都走远了")
end)
```

### 双阈值滞回机制

`SetDist(near, far)` 设置了两个同心圆。near < far，中间有缓冲区，防止玩家在边界反复横跳时事件疯狂触发：

```
         near=8     far=10
           +--------+
          /          \
         /    +--+    \
        |     |  |     |
        |     |  |     |   进入 8 内 → 触发 onnear
        |     +--+     |
        |              |   离开 10 外 → 触发 onfar
         \            /
          \          /
           +--------+
```

这个设计很重要——如果你自己用 `DoPeriodicTask` + `IsNear()` 手写，没有滞回的话玩家在 9.9~10.1 之间反复跨越会炸。

### 四种目标模式

`playerprox` 构造时可选模式，默认 `AnyPlayer`：

```lua
-- 构造时指定模式（一般不直接调构造，通过 SetTargetMode 切换）
-- 默认 AnyPlayer：任意玩家进入 near 就触发 onnear，所有玩家退出 far 才触发 onfar

-- AllPlayers：每个玩家独立追踪进入/离开
-- inst.components.playerprox:SetTargetMode(PlayerProx.TargetModes.AllPlayers)

-- SpecificPlayer：只追踪指定玩家
-- inst.components.playerprox:SetTargetMode(PlayerProx.TargetModes.SpecificPlayer, target_player)

-- LockOnPlayer：第一个靠近的玩家被锁定，追踪到他离开为止
-- inst.components.playerprox:SetTargetMode(PlayerProx.TargetModes.LockOnPlayer)

-- LockAndKeepPlayer：锁定第一个靠近的玩家并永远追踪他
-- inst.components.playerprox:SetTargetMode(PlayerProx.TargetModes.LockAndKeepPlayer)
```

模式选择一览：

| 模式 | onnear 触发 | onfar 触发 | 适用场景 |
|------|------------|-----------|----------|
| AnyPlayer（默认） | 第一个人进入 | 最后一个人离开 | 灯光、机关等"有人就激活" |
| AllPlayers | 每个玩家进入 | 每个玩家离开 | 需要区分具体玩家的逻辑 |
| SpecificPlayer | 指定玩家进入 | 指定玩家离开 | NPC 跟随特定玩家 |
| LockOnPlayer | 第一个人进入 | 被锁定的人离开 | 自动售货机等"谁先来服务谁" |
| LockAndKeepPlayer | 第一个人进入 | 不触发（永锁） | 专属绑定物品 |

### 其他可选设置

```lua
-- 限制只对活着的玩家响应（幽灵不算）
inst.components.playerprox:SetPlayerAliveMode(PlayerProx.AliveModes.AliveOnly)
-- 可选: AliveOnly / DeadOnly / DeadOrAlive（默认）

-- 设置检测频率（默认 10 帧 ≈ 0.33 秒，更频繁=更灵敏但更耗性能）
-- inst.components.playerprox:Schedule(5 * FRAMES)  -- 每 5 帧检测一次

-- 主动查询当前是否有玩家在范围内
if inst.components.playerprox:IsPlayerClose() then
    -- 有人
end

-- 强制立即检测一次（不等待下一次定时器）
inst.components.playerprox:ForceUpdate()
```

### 典型示例：档案室吊灯（archive_chandelier.lua）

```lua
-- 靠近 20 亮灯，离开 23 灭灯
local function updatelight(inst)
    local playerprox = inst.components.playerprox
    local powered = (playerprox == nil or playerprox:IsPlayerClose())
        and (archivemanager == nil or archivemanager:GetPowerSetting())
    if powered then
        -- 亮灯逻辑 ...
    else
        -- 灭灯逻辑 ...
    end
end

-- 注册
inst:AddComponent("playerprox")
inst.components.playerprox:SetDist(20, 23)  -- 双阈值
inst.components.playerprox:SetOnPlayerNear(updatelight)
inst.components.playerprox:SetOnPlayerFar(updatelight)
```

### 典型示例：暗影之眼（creepyeyes.lua）

```lua
-- 玩家靠近 3 距离 → 消失，离开 5 距离 → 出现
-- 注意：和吊灯相反——靠近触发"消除"而非"激活"
inst:AddComponent("playerprox")
inst.components.playerprox:SetDist(3, 5)
inst.components.playerprox:SetOnPlayerNear(Disappear)  -- 靠近 → 消失！
```

## DIY 方案：手写距离检测

当 `playerprox` 不够灵活时——比如你需要检测**非玩家实体**、需要自定义触发条件、或者要同时扫描多种实体——可以手写。

### 核心 API

所有 API 的 `isalive` 参数控制是否过滤死亡/幽灵玩家：
- `nil` — 死人都算（默认）
- `true` — 只算活人
- `false` — 只算死人/幽灵

```lua
-- 最常用：找范围内最近的玩家（不含距离返回）
local player = FindClosestPlayerToInst(inst, 15, true)
if player ~= nil then
    -- 15 范围内有活着的玩家，player 是最近的那个
end

-- 判断任意玩家是否在范围内（布尔，性能优于 FindPlayersInRange）
if IsAnyPlayerInRange(x, y, z, 10, true) then
    -- 有活着的玩家在 10 范围内
end

-- 获取范围内所有玩家列表
local players = FindPlayersInRange(x, y, z, 8, nil)
for i, player in ipairs(players) do
    -- 处理每个玩家
end

-- 实体自身的距离判断方法（定义在 entityscript.lua）
local dist = 10
if inst:IsNear(target, dist) then
    -- target 在 inst 的 10 距离内（水平距离，忽略 y 高度）
end

if inst:IsNearPlayer(12, true) then
    -- 有活着的玩家在 12 距离内
end
```

### 眼球塔示例：战斗目标选取（eyeturret.lua + eyeturretbrain.lua）

眼球塔同时用了两条路径——combat 用 `TheSim:FindEntities()` 做范围扫描，brain 用 `FindClosestPlayerToInst()` + `IsNear()` 做视线跟随：

```lua
-- eyeturret.lua: retargetfn — 用 TheSim:FindEntities 扫描战斗目标
local RETARGET_MUST_TAGS = { "_combat" }
local RETARGET_CANT_TAGS = { "INLIMBO", "player", "eyeturret", "engineering" }

local x, y, z = inst.Transform:GetWorldPosition()
local ents = TheSim:FindEntities(x, y, z, TUNING.EYETURRET_RANGE + 3,
    RETARGET_MUST_TAGS, RETARGET_CANT_TAGS)
-- TheSim:FindEntities 返回圆形范围内的所有实体，可按 tag 过滤
-- 参数: x, y, z, 半径, must_tags(表/单值/nil), cant_tags, has_any_tags

-- shouldKeepTarget — 用 inst:IsNear 判断是否还在攻击范围内
local function shouldKeepTarget(inst, target)
    return inst.components.combat:CanTarget(target)
        and inst:IsNear(target, TUNING.EYETURRET_RANGE + 3)
end

-- eyeturretbrain.lua: 用 FindClosestPlayerToInst + IsNear 控制视线朝向
local START_FACE_DIST = 10
local KEEP_FACE_DIST = 15

local function GetFaceTargetFn(inst)
    local target = FindClosestPlayerToInst(inst, START_FACE_DIST, nil)
    return target ~= nil and not target:HasTag("notarget") and target or nil
end

local function KeepFaceTargetFn(inst, target)
    return not target:HasTag("notarget") and inst:IsNear(target, KEEP_FACE_DIST)
end
-- GetFaceTargetFn: 玩家进入 10 开始面向他
-- KeepFaceTargetFn: 玩家超出 15 才放弃面向
-- 这就是手动实现的双阈值滞回
```

## 两条路径的性能对比

| | playerprox | 手写 DoPeriodicTask + IsNear |
|---|---|---|
| 代码量 | 3~5 行 | 15~30 行（需要自己管理定时器、状态、滞回） |
| 性能 | 良好（10 帧间隔、轻量 API） | 取决于实现 |
| 灵活性 | 仅"玩家"进出 | 任意实体、任意条件 |
| 滞回 | 内置 | 需自己实现 |
| 休眠支持 | 自动（OnEntitySleep 停/ OnEntityWake 启） | 需自己处理 |
