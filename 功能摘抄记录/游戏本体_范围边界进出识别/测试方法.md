# 测试方法

## 测试 Mod 骨架

要验证本功能摘抄中的 API，可以搭建一个简单的测试 Mod：

```
hotreload_test/
├── modinfo.lua
└── modmain.lua
```

### modinfo.lua 最低要求

```lua
name = "范围检测测试"
description = "验证 playerprox 和 DIY 距离检测"
author = "测试者"
version = "1.0"
api_version = 10
dst_compatible = true
all_clients_require_mod = false  -- 仅本地测试
client_only_mod = false

configuration_options = {
    -- 可选：加一个选项控制测试目标（axe / spear / 自定义物品）
}
```

### 建议配合的 Mod

- **Insight (Show Me+)** — Steam 创意工坊 ID `2189004162`，可在鼠标悬停时显示物品数值、组件信息，便于观察距离检测是否正确触发
- 控制台命令辅助：`c_give("axe")` 生成测试物品，`c_spawn("spider")` 生成测试生物

## 操作步骤与预期结果

### 测试 1：playerprox — 任意玩家靠近触发

1. 启用测试 Mod，进入世界
2. 在控制台输入生成测试物品
3. 远离该物品（超过 far 距离）
4. 走向该物品，进入 near 距离
5. 观察触发效果（声音/光效/聊天框消息）

**预期**：进入 near 距离时触发 onnear 回调，远离超过 far 距离时触发 onfar 回调。

### 测试 2：playerprox — 双阈值滞回

1. 在边界（near~far 之间）反复进出
2. 观察是否每次跨越都触发事件

**预期**：进入 near 以内才触发 onnear，退出 far 以外才触发 onfar；在 near~far 缓冲区内不应反复触发。

### 测试 3：DIY 方案 — IsNear + 手动轮询

1. 启用使用了 `inst:IsNear(target, dist)` 的自定义 Prefab
2. 让目标实体进入/离开检测范围
3. 观察是否按预期触发

**预期**：与 playerprox 效果一致，但行为更可控（可以自定义轮询频率和目标类型）。

### 测试 4：isalive 参数 — 过滤死亡玩家

1. 让一个玩家角色死亡（变成幽灵）
2. 用幽灵形态走进检测范围
3. 观察是否触发

**预期**：如果 `isalive = true`，幽灵不应触发检测；如果 `isalive = nil`（默认），幽灵也会触发。
