# 查找范例

## 文件用途速览

| 文件 | 作用 | 大小 |
|------|------|------|
| `prefablist.lua` | 所有已注册 Prefab 的 ID 列表（按字母排序） | ~1,500 行 |
| `tuning.lua` | 全局数值参数表（血量、伤害、距离、时长……） | ~9,500 行 |
| `chinese_s.po` | Prefab ID → 中文名映射（标准 gettext PO 格式） | ~17MB |

---

## 范例 1：prefablist.lua — 查某个 Prefab 是否存在

**问**：游戏里有没有"暗影棋子"这个 Prefab？

**查法**：在 `prefablist.lua` 中搜索 `shadow`。

```lua
-- 文件头部就是字母序的完整列表：
PREFABFILES = {
  "abigail",
  "abigail_attack_fx",
  ...
  "shadow_bishop",      -- 暗影主教
  "shadow_hand",        -- 暗影之手
  "shadow_knight",      -- 暗影骑士
  "shadow_rook",        -- 暗影战车
  ...
}
```

找到说明该 Prefab 已注册，可以用于 `AddPrefabPostInit("shadow_knight", fn)` 等操作。

**注意**：`PREFABFILES` 只是文件名列表，实际脚本路径是 `scripts/prefabs/{名称}.lua`。

## 范例 2：tuning.lua — 查某个数值参数

**问**：斧子的基础伤害是多少？

**查法**：在 `tuning.lua` 中搜索 `AXE_DAMAGE`。

```lua
-- 第 357 行：
AXE_DAMAGE = wilson_attack * 0.8,
```

再追 `wilson_attack`：

```lua
-- 第 354 行：
SPEAR_DAMAGE = wilson_attack,
-- wilson_attack 定义在更前面，约第 110 行附近：
WILSON_ATTACK = 34,
```

所以 `AXE_DAMAGE = 34 * 0.8 = 27.2`。

**在 Mod 中修改**：
```lua
-- modmain.lua 顶层（改了之后新世界生效）
TUNING.AXE_DAMAGE = 27.2 * 2  -- 翻倍
```

**其他常用参数查法示例**：

| 搜索关键词 | 含义 | 实际值 |
|-----------|------|--------|
| `WILSON_HEALTH` | Wilson 基础血量 | `150` |
| `WILSON_RUN_SPEED` | Wilson 跑步速度 | `6` |
| `INSULATION_LARGE` | 高级保温时长 | `seg_time * 8` = 240 秒 |
| `PERISH_FRIDGE_MULT` | 冰箱保鲜倍率 | `0.5` |
| `TOTAL_DAY_TIME` | 白天段总长 | `seg_time * 16` |
| `EYETURRET_RANGE` | 眼球塔射程 | （具体值查该行） |
| `AXE_USES` | 斧子耐久 | （具体值查该行） |

## 范例 3：chinese_s.po — 查 Prefab ID 对应的中文名

**问**：`"axe"` 的中文名叫什么？

**查法**：在 `chinese_s.po` 中搜索 `STRINGS.NAMES.AXE`。

```po
#. STRINGS.NAMES.AXE
msgctxt "STRINGS.NAMES.AXE"
msgid "Axe"
msgstr "斧头"
```

答案：**斧头**。

**PO 格式解读**：
- `msgctxt` — 上下文，指明这是哪个 STRINGS 路径下的条目（用于区分同名物品）
- `msgid` — 英文原文（与 `strings.lua` 中的 `STRINGS.NAMES.XXX` 值对应）
- `msgstr` — 简体中文翻译

**另一个例子**：查 `"townportal"`：

```po
#. STRINGS.NAMES.TOWNPORTAL
msgctxt "STRINGS.NAMES.TOWNPORTAL"
msgid "The Lazy Deserter"
msgstr "懒人魔杖"
```

可以看到英文名 "The Lazy Deserter" 和中文名 "懒人魔杖" 完全不同——所以不能靠英文名猜测中文名，必须以 PO 文件为准。

**完整查询链**：

```
需要改某个物品 → 知道它的 Prefab ID（如 "axe"）
    ↓ 在 strings.lua 或 .po 的 msgctxt 中找到它的英文名
    "Axe"
    ↓ 在 chinese_s.po 中找 msgid "Axe" 对应的 msgstr
    "斧头"
```

如果你只知道中文名不知道 Prefab ID：在 `chinese_s.po` 中搜 `msgstr "斧头"`，往上翻几行看 `msgctxt` 里写的 `STRINGS.NAMES.XXX`，`XXX` 就是 Prefab ID。
