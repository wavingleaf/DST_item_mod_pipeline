# DST Mod 基本结构

## 标准目录结构

```
my-mod/                          -- 你的 mod 根文件夹，命名随意（建议全小写无空格）
│
├── modinfo.lua                  -- [会被框架检索] 必须存在。mod 元信息：名称、版本、
│                                    作者、描述、api_version、配置选项等。
│                                    启动时由游戏框架自动读取并注册 mod。
│
├── modmain.lua                  -- [会被框架检索] 服务端 + 客户端（双端）都执行的
│                                    逻辑。游戏加载 mod 时自动执行该文件。通常在此
│                                    注册自定义物品、角色、配方、组件钩子等。
│
├── modworldgenmain.lua          -- [会被框架检索] 可选。世界生成相关逻辑。仅在
│                                    世界生成阶段执行，用于修改地形、生物群落、
│                                    资源分布等。
│
├── modicon.png                  -- [会被框架检索] 可选但强烈建议。mod 列表界面
│                                    显示的图标。推荐 256×256 像素，PNG 格式。
│                                    没有此文件将显示默认占位图标。
│
├── modicon.tex                  -- [会被框架检索] 可选。modicon.png 的 TEX 转换
│                                    文件（由游戏工具自动生成）。如果只提供 png，
│                                    游戏会在首次加载时自动转换。
│
├── modicon.xml                  -- [会被框架检索] 可选。modicon.tex 的 XML 描述
│                                    文件（由游戏工具自动生成）。
│
├── preview.png                  -- [会被框架检索] 可选但强烈建议。创意工坊/Mod
│                                    详情页的预览图。没有严格尺寸限制，建议
│                                    16:9 宽高比。
│
├── LICENSE                      -- 可选。开源许可证文件。
│
├── README.md                    -- 可选。说明文档。
│
├── exported/                    -- 存放由官方工具导出的动画/贴图资源
│   ├── anim/                    --   动画文件 (.zip → 游戏内动画资源)
│   └── images/                  --   贴图文件 (.xml + .tex)
│
├── anim/                        -- 动画资源（与 exported/ 类似，手动的）
├── images/                      -- 贴图资源
├── sound/                       -- 音频资源 (.fsb 格式)
├── scripts/                     -- Lua 脚本文件夹
│   ├── components/              --   自定义组件
│   ├── prefabs/                 --   自定义实体（物品、生物、建筑等）
│   ├── stategraphs/             --   状态图（动画状态机）
│   ├── brains/                  --   生物 AI 逻辑
│   ├── screens/                 --   UI 界面
│   ├── widgets/                 --   UI 控件
│   ├── map/                     --   地图相关
│   └── tools/                   --   工具函数
│
├── strings/                     -- 本地化/翻译文件（如果有多语言需求）
│   ├── chinese.po               --   简体中文翻译
│   └── english.po               --   英文翻译
│
├── bigportraits/                -- 选人界面的角色大图
├── save/                        -- mod 存档数据文件夹
└── anim/                        -- 动画资源
```

## 会被框架自动检索的特殊文件名

以下文件名由 DST mod 框架在特定阶段自动查找并加载，不能随意改名：

| 文件名 | 是否必需 | 说明 |
|--------|----------|------|
| `modinfo.lua` | **必需** | mod 的身份证，没有它将无法被游戏识别 |
| `modmain.lua` | **必需** | mod 主逻辑入口，双端执行 |
| `modworldgenmain.lua` | 可选 | 世界生成入口，仅服务端执行 |
| `modicon.png` | 推荐 | mod 列表图标 |
| `modicon.tex` | 可选 | png 的纹理压缩格式，由工具生成 |
| `modicon.xml` | 可选 | tex 的坐标描述，由工具生成 |
| `preview.png` | 推荐 | 创意工坊预览图 |

> `scripts/` 文件夹下的文件名**没有**特殊含义，你可以自由组织。但按照社区惯例，`components/`、`prefabs/` 等子文件夹名暗示了内容类型。

## 其他需要注意的事项

- 所有 Lua 脚本中的文件引用路径是相对于 mod 根目录的。
- `api_version` 决定你的 mod 兼容的游戏版本范围，当前常用 `api_version = 10`。
- 如果需要兼容服务端-only mod（客户端不需要订阅），设置 `all_clients_require_mod = false` 并额外提供 `modoverrides.lua` 说明项。
