# 音频相关

## FMOD Designer ≠ FMOD Studio——DST 只认旧版 FMOD Designer

### 现象

以为用 FMOD Studio（新版音频工具）导出 `.bank` 格式就行，实际上游戏毫无反应。

### 根因

DST 的音频引擎基于 **FMOD Designer**（2011 年停更的旧版工具），只支持 `.fev` + `.fsb` 格式。FMOD Studio 导出的是 `.bank` 格式，**与 DST 完全不兼容**。Klei 开发者曾在论坛明确说过 FMOD Studio 出现在 mod 工具里是「意外」。

### 正确工具

**FMOD Designer**，由 Klei 官方通过 Steam 免费提供，内置于 **Don't Starve Mod Tools**：

1. Steam 库 → 左上角筛选下拉 → 勾选「工具」→ 搜索 "Don't Starve Mod Tools" → 安装
2. 安装后路径：`Steam\steamapps\common\Don't Starve Mod Tools\mod_tools\FMOD_Designer\`
3. GUI：`fmod_designer.exe`
4. 命令行构建：`fmod_designercl.exe`

只要拥有 Don't Starve 即可免费使用，**不需要**去第三方网站下载 FMOD Designer。

### 命令行构建命令

```bash
fmod_designercl.exe -pc -b <输出目录> <project.fdp>
```

- `-pc`：目标平台（必选）
- `-b <path>`：输出目录
- `-a <path>`：可选的音频源目录覆盖

## FMOD Designer CLI（fmod_designercl.exe）不兼容中文路径

### 现象

命令行构建时 `.fev` 能生成，但 `.fsb` 构建失败，报错：

```
Cannot create directory 'D:\Github??\mod???\...\sound\.fsbcache\'
```

### 根因

`fmod_designercl.exe` 是 2011 年的 32 位 Windows 程序，内部路径处理不支持 UTF-8/Unicode。中文路径会被编码为 `?` 乱码，导致 `.fsbcache` 临时目录创建失败，进而 `.fsb` 构建整体失败。

`.fev` 的生成不受影响（不需要临时目录），但 `.fsb` 是实际音频数据文件，缺了它 Mod 不会有声音。

### 修复

用 `subst` 将含中文的 mod 目录映射为纯 ASCII 盘符后构建：

```bash
# 映射
subst X: "d:/含中文路径的mod目录"

# 构建
fmod_designercl.exe -pc -b "X:/sound" "X:/sound/project.fdp"

# 构建完成后解除映射
subst X: /d
```

`subst` 是 Windows 内置命令（DOS 时代就有了），不需要管理员权限，重启后自动清除。

同理，`.fdp` 项目文件内的 **波形引用路径和音频源目录路径** 也必须是纯 ASCII，否则即使 `subst` 了输出路径，读取 WAV 源文件时仍会中文乱码。

## FMOD Designer 事件路径格式

### 格式

```
<.fdp 项目名>/<Group 名>/<Event 名>
```

### 示例

```
chen_alarm/sound/dangdang
```

- `chen_alarm` — `.fdp` 项目文件名（不含扩展名）
- `sound` — Group 名（**全小写，固定命名**，DST 社区 Mod 约定）
- `dangdang` — Event 名（自定义，与 `.fdp` 中创建的名字完全一致）

### 教训

- Group 必须重命名为 `sound`，不能保留默认名
- `.fdp` 文件名决定了 SoundEmitter 中 `PlaySound` 的路径第一段，建议与 Bank 名一致
- 小穹 mod 的参考格式：`"sora/sound/talk_LP"`（项目名/组名/事件名）

## Bank 名决定输出文件名

### 教训

FMOD Designer 中的 Bank 名直接决定导出文件的文件名：

- Bank 名 `chen_alarm` → 导出 `chen_alarm.fev` + `chen_alarm.fsb`
- Bank 名 `chen_alarm_bank01` → 导出 `chen_alarm_bank01.fev` + `chen_alarm_bank01.fsb`

在 `modmain.lua` 中注册的 Assets 路径和 Bank 名必须一致：

```lua
Assets = {
    Asset("SOUNDPACKAGE", "sound/chen_alarm.fev"),   -- ← 必须与 Bank 名一致
    Asset("SOUND", "sound/chen_alarm.fsb"),          -- ← 必须与 Bank 名一致
}
```

如果 FMOD Designer 中创建 Bank 时用了默认名（如 `chen_alarm_bank01`），要么改代码，要么改 Bank 名。建议在 `.fdp` 中直接搜索替换统一。

## `.fsbcache` 临时目录——发布前必须删除

### 现象

Steam Workshop 上传工具报错或拒绝上传。

### 根因

FMOD Designer 构建时会在输出目录生成 `.fsbcache` 临时文件夹。Workshop Uploader 不认识这个文件夹，会导致上传失败。

### 修复

构建完成后立即删除：

```bash
rm -rf sound/.fsbcache
```

## WAV 格式要求

### 经过验证的参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 编码 | PCM signed 16-bit LE | 无损，FMOD Designer 内编码为 FSB |
| 采样率 | 44100 Hz | 标准 CD 音质 |
| 声道 | 2（立体声） | DST 引擎支持立体声 |
| 转换命令 | `ffmpeg -i in.mp3 -acodec pcm_s16le -ar 44100 -ac 2 out.wav` | 从 MP3 转 |

FMOD Designer 导入 `.wav` 后在构建时自行编码为目标平台格式写入 `.fsb`。

## FMOD Designer GUI 保存时会损坏路径 → 尽量不用 GUI

### 现象

在 FMOD Designer GUI 中打开 `.fdp`，关闭时提示保存（即使"没改任何东西"也会触发保存），之后命令行构建出的 `.fsb` 只有 192 字节（正常应该是数百 KB），游戏里完全没有声音。

```
-rw-r--r-- 1 yk liu 197609 553536 Jul  8 02:14 chen_alarm.fsb   ← 正常
-rw-r--r-- 1 yk liu 197609    192 Jul  8 02:48 chen_alarm.fsb   ← 被 GUI 保存后构建：静默空壳
```

### 根因

`fmod_designercl.exe` 和 `fmod_designer.exe`（GUI）对路径的编码方式不同：

- **CLI 工具**：需要纯 ASCII 路径，含中文会乱码，但至少会报错
- **GUI**：是 GBK 时代的程序，保存项目时会把 `_PC_audiosourcedirectory` 以 GBK 编码写入 `.fdp`。中文路径变成 `D:/Github??/mod???/...` 乱码

之后用 CLI 构建时，`fmod_designercl.exe` 读到这个乱码路径，找不到 WAV 文件，**不报任何错误**，静默生成一个只有头骨架的 192 字节空 `.fsb`。构建日志中的 `SUCCESS` 是靠不住的——必须看 `.fsb` 文件大小。

更隐蔽的是：**GUI 会自动保存**。关掉窗口时，即使你只是翻看、没改任何东西，它也可能提示保存——点了「是」，路径就被写坏了。FMOD Designer 还有一个 `.fdp.autosave` 文件，说明它会在后台自动保存。

### 策略：尽可能避免用 GUI，交给命令行处理

**GUI 只用来做一件事：首次创建项目**（新建 `.fdp`、创建 Group、创建 Event、添加 WAV 引用、创建 Bank）。之后所有操作——改音量、改路径、构建、重建——全部用命令行和文本编辑完成。

**操作原则**：

| 操作 | 用什么 | 命令/手段 |
|------|--------|-----------|
| 首次创建项目 | GUI（必须，CLI 无创建能力） | `fmod_designer.exe` |
| 查看项目结构 | 文本编辑器 | 直接读 `.fdp`（它本质是 XML 文本） |
| 修改音量 | sed / 文本编辑 | `sed -i 's|<volume>1</volume>|<volume>0.67</volume>|g' project.fdp` |
| 修改音频源目录 | sed | 见下方修复命令 |
| 修改 Bank 名 | sed | `sed -i 's|old_bank_name|new_bank_name|g' project.fdp` |
| 构建导出 | CLI | `fmod_designercl.exe -pc -b "Q:/sound" "Q:/sound/project.fdp"` |

### 如果不得不用了 GUI 怎么办

GUI 关掉后，**必须立即修复 `.fdp` 中被写坏的路径**，再构建：

```bash
# 修复音频源目录（被 GUI 以 GBK 编码写入，变成乱码）
sed -i 's|<_PC_audiosourcedirectory>.*</_PC_audiosourcedirectory>|<_PC_audiosourcedirectory>D:/no_chinese_char_proj/wav_folder</_PC_audiosourcedirectory>|' project.fdp

# 修复构建输出目录
sed -i 's|<_PC_builddirectory>.*</_PC_builddirectory>|<_PC_builddirectory>Q:/sound</_PC_builddirectory>|' project.fdp

# 再构建
fmod_designercl.exe -pc -b "Q:/sound" "Q:/sound/project.fdp"
```

**注意**：这条修复命令对 GUI 创建/删除 Event、修改 Bank 等操作无影响——那些是项目结构数据，不会被路径乱码牵连——所以只要你关掉 GUI 后修了路径，`.fev`/`.fsb` 就能正常构建。

### 构建后必须检查 `.fsb` 文件大小

```bash
ls -la project.fsb
```

- 正常：几百 KB（3 段短音频约 500~600 KB）
- 空壳：192 字节 → **音频源目录乱码，WAV 未载入，构建失败但未报错**

不要只管构建日志里的 `SUCCESS`，那个不可靠。

## 构建流程速查

```bash
# 1. 准备 WAV（纯 ASCII 路径，无中文）
# 2. 用 FMOD Designer GUI 创建 .fdp 项目（仅此一次）
# 3. GUI 关掉后立刻修复路径
sed -i 's|<_PC_audiosourcedirectory>.*</_PC_audiosourcedirectory>|<_PC_audiosourcedirectory>D:/path/to/wav</_PC_audiosourcedirectory>|' sound/project.fdp
# 4. 用 subst 映射路径（如需）
subst X: "d:/my_mod"
# 5. 命令行构建
fmod_designercl.exe -pc -b "X:/sound" "X:/sound/project.fdp"
# 6. 检查产物大小（防静默失败）
ls -la X:/sound/project.fsb   # 必须 > 100 KB，192 bytes 就是挂了
# 7. 清理
rm -rf X:/sound/.fsbcache
subst X: /d
```

**后续修改**（音量、音频文件等）：**不要打开 GUI**。直接 `sed` 改 `.fdp` 中的对应值，然后跑步骤 4-7 重构建。`.fdp` 是纯文本 XML，所有参数都可以命令行修改。

## FMOD Designer 默认 2D 模式——声音全图满音量播放

### 现象

闹钟触发时，无论玩家离闹钟多远音量完全相同，没有远近衰减、没有左右声道偏转。

### 根因

FMOD Designer 创建 Simple Event 时的默认模式是 `x_2d`（2D 全局模式）。此模式下音频不参与 FMOD 的 3D 空间计算，`maxdistance=10000` 意味着声音覆盖整张地图。

### 修复

将 Event 的 mode 从 `x_2d` 改为 `x_3d`，并设置合理的衰减距离：

```bash
sed -i 's|<mode>x_2d</mode>|<mode>x_3d</mode>|g' project.fdp
sed -i 's|<mindistance>1</mindistance>|<mindistance>3</mindistance>|g' project.fdp
sed -i 's|<maxdistance>10000</maxdistance>|<maxdistance>30</maxdistance>|g' project.fdp
```

| 参数 | 之前 | 之后 | 含义 |
|------|------|------|------|
| `mode` | `x_2d` | `x_3d` | 从全局平面 → 空间音效 |
| `mindistance` | `1` | `3` | 3 个地皮宽度内满音量 |
| `maxdistance` | `10000` | `30` | 3~30 单位线性衰减，30 外静音 |

DST 的 `SoundEmitter` 自动将实体世界坐标传给 FMOD，3D 模式下引擎据此计算距离衰减和左右声道偏转。

### 为什么 GUI 创建时默认是 2D

FMOD Designer 的 `Add Event → Simple` 模板不含 3D 属性。如果要在创建时就有 3D，应该用 `Add Event → Simple with 3D Properties`。但更简单的做法是直接用 Simple 创建，事后用 sed 改 `.fdp`。

## 音量调参经验

### 迭代过程

小陈闹钟的音量从默认 `1` 开始，共经历 5 轮调整：

| 轮次 | volume | 同时修改 | 效果 | 学到啥 |
|------|--------|----------|------|--------|
| 1 | 0.67 | — | 没变化 | GUI 关掉时悄悄保存，把 volume 覆盖回了 1 |
| 2 | 0.4 | — | 完全没声音 | GUI 把 audio source 路径写成了中文乱码，`.fsb` 变成 192 字节空壳 |
| 3 | 1 | 修复路径重新构建 | 正常 | 验证了修复命令有效 |
| 4 | 0.5 | — | 偏大 | — |
| 5 | 0.2 | 同时切到 3D 模式 | 偏小，但 3D 有距离感了 | — |
| **终** | **0.4** | 3D 模式 | 正常 | 近处清晰、走远自然淡出 |

### 教训

1. **音量和 3D 模式不可分拆调**。2D 模式下音量仅受 volume 控制；3D 模式下 mindistance/maxdistance 也参与实际听感——同样的 volume=0.4，2D 模式下全图满响，3D 模式下站近才听得清
2. **一次只改一个变量**：要么改 volume，要么改距离参数，混着改分不清哪个在起作用
3. 调参全程不要开 GUI——每次 `sed → CLI构建 → 进游戏听`，流程干净，不会被 GUI 的自动保存干扰
