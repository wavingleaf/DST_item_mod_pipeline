# 配方物品相关

## `AddRecipe2` 的 `placer` 参数触发结构物建造模式

### 现象

制作便携衣柜后产物直接部署到了地面上，捡不起来（没有 `inventoryitem` 组件）。

### 根因

`AddRecipe2` 的 `placer` 参数不是"预览模型"那么简单。**只要填了 `placer`，框架就认为这个配方的产物是"结构物"（structure）而非"物品"**，进入"先建造再摆放"的流程——产物直接出现在地面上，不经过物品栏。

这是 DST 配方系统的底层逻辑：`placer` ≠ 可选的预览美化，而是 `DEPLOYMODE` 的开关：

- 有 `placer` → 结构物模式，产物不走物品栏
- 无 `placer` → 物品模式，产物进入物品栏

### 正确做法

如果你的物品是通过自己的 `deployable` 组件实现"从物品栏右键部署到地面"，**不要在配方里填 `placer`**。`placer` 只给那些"造出来就是建筑"的东西用（如科学机器、箱子、原版衣柜）。

```lua
-- ❌ 物品栏版 + placer → 框架走结构物建造流程，产物直接在地面
AddRecipe2("portable_wardrobe_ly", {...}, TECH.NONE,
    { numtogive = 1, placer = "portable_wardrobe_ly_placer" })

-- ✅ 物品栏版 + 无 placer → 产物进物品栏，由 deployable 组件负责部署
AddRecipe2("portable_wardrobe_ly", {...}, TECH.NONE,
    { numtogive = 1 })
```

### 补充：配方栏图标的 `atlas`/`image`

- 在 mod 加载时通过 `resolvefilepath` 解析，只搜索 mod 自身目录
- 不能直接引用原版路径（如 `images/inventoryimages.xml`），除非文件已复制到 mod 目录下
- 调测阶段 `nil` 表现为空白图标，非致命
