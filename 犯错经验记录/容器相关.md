# 容器相关

## `usespecificslotsforitems` + `GetSpecificSlotForItem` 的一对多匹配缺陷

### 现象

物品可匹配多列（如既有保暖又有防水的装备），第一匹配槽被占后直接失败，不尝试后续匹配槽。

### 根因

`container.lua:298-303` 的 `GetSpecificSlotForItem` 只返回**第一个 itemtestfn 通过的槽**，不检查是否已被其他物品占据：

```lua
function Container:GetSpecificSlotForItem(item)
    if self.usespecificslotsforitems and not self.readonlycontainer and self.itemtestfn ~= nil then
        for i = 1, self:GetNumSlots() do
            if self:itemtestfn(item, i) then
                return i          -- 匹配，但可能已被占 ← 缺陷
            end
        end
    end
end
```

`GiveItem`（`container.lua:523`）在 `usespecificslotsforitems = true` 时跳过"任意空槽"兜底逻辑。普通箱子不受影响——它们没有 `usespecificslotsforitems`，不走此函数，直接用"第一个空槽"兜底。

### 修复

在 Prefab 构造阶段覆盖此方法，加 `slots[i] == nil` 检查：

```lua
inst.components.container.GetSpecificSlotForItem = function(self, item)
    if self.usespecificslotsforitems and not self.readonlycontainer and self.itemtestfn ~= nil then
        for i = 1, self:GetNumSlots() do
            if self:itemtestfn(item, i) and self.slots[i] == nil then
                return i          -- 匹配 且 空闲
            end
        end
    end
end
```

## `itemtestfn` 的 `slot` 参数可能为 `nil`

### 现象

把物品放入容器时崩溃：
```
attempt to compare nil with number
```

### 根因

两条调用路径进入 `itemtestfn`，slot 的情况不同：

**路径 A — "快速否决"（slot = nil）**：`actions.lua:2491`

```lua
elseif not target.components.container:CanTakeItemInSlot(act.invobject) then
    -- 只传 item 不传 slot，问"这容器能否接受这种物品（不管放哪个槽）？"
    return false, "NOTALLOWED"
end
```

**路径 B — "逐槽匹配"（slot = 具体数字）**：`container.lua:406`，正常。

框架自己的 `CanTakeItemInSlot`（`container.lua:280`）对 nil 做了 guard，但最后一行把 slot **原样透传**给自定义 `itemtestfn`：

```lua
return self.itemtestfn == nil or self:itemtestfn(item, slot)  -- slot 可能为 nil
```

### 修复

`itemtestfn` 第一行先判空：

```lua
local function itemtestfn(container, item, slot)
    if not slot then
        -- 框架在"快速否决"路径，不涉及具体槽位。
        -- 遍历所有槽，任一匹配即可。
        local n = container._numslots or container.numslots  -- 兼容客户端 replica
        for i = 1, n do
            if CheckSlotMatch(item, i) then
                return true
            end
        end
        return false
    end
    -- 之后再做 slot 相关的数值比较
end
```

注意：`container` 在服务端是 `container` 组件（有 `numslots`），在客户端是 `container_replica`（用 `_numslots`），需要双路径兼容。

## `containers.params` key 必须与 Prefab 名一致

`container:WidgetSetup("prefab_name")` 内部做 `params[prefab_name]` 查找。因此：

- Prefab 名、`WidgetSetup` 参数、`containers.params` 的 key，三者必须完全一致
- 改了 Prefab 文件名要同步改所有引用处

如果 key 不匹配，容器将使用默认参数（可能是错误的 slotpos / animbank / itemtestfn），或不显示 UI。
