# 物品相关

## 图集文件的两道独立检查：`resolvefilepath` + `Asset()` 声明

### 问题链条

配方栏和物品栏图标需要图集文件（`.tex` + `.xml`）。以下三次尝试分别失败了，最终揭示了两套独立系统：

#### 尝试 1：recipe 直接填原版路径

```lua
AddRecipe2(..., { atlas = "images/inventoryimages.xml", image = "wardrobe.tex" })
```

**结果**：mod 加载阶段 `resolvefilepath` 报错 —— 文件不在 mod 目录中。

**原因**：`resolvefilepath` 遍历 `package.assetpath` 搜索路径，mod 目录在搜索路径中，但前提是**文件确实放在 mod 目录下**。

#### 尝试 2：把 `.tex` 和 `.xml` 复制到 `mod/images/` 下

**结果**：`resolvefilepath` 通过了，但运行时 `Invalid resource handle for atlas`。

**原因**：`resolvefilepath` 只查文件系统存在性。引擎的**纹理资源句柄系统**是另一层——它只认通过 `Asset()` 函数在 Prefab 的 assets 表中显式声明过的文件。没声明 = 引擎不知道这个图集存在。

#### 尝试 3（成功）：文件存在 + `Asset()` 声明

```lua
local assets = {
    Asset("ATLAS", "images/inventoryimages.xml"),
}
```

**结论**：两道检查**缺一不可**：

| 检查 | 时机 | 负责函数 | 失败后果 |
|------|------|----------|----------|
| 文件存在 | Mod 加载时 | `resolvefilepath` → `kleifileexists` | 崩溃 |
| Asset 声明 | 运行时加载纹理 | 引擎资源句柄系统 | `Invalid resource handle`，图标不显示 |

## `inventoryitem` 的图标通过 net_var 属性设置，而非方法调用

### 现象

```lua
inst.components.inventoryitem:SetAtlas("images/inventoryimages.xml")  -- nil crash
```

崩溃：`attempt to call method 'SetAtlas' (a nil value)`。

### 根因

`inventoryitem` 组件的服务端实现**没有** `SetAtlas` / `SetImage` 方法。图标信息通过 net_var **属性** `atlasname` / `imagename` 传递，赋值触发回调推送到客户端 replica：

```lua
-- inventoryitem.lua（组件内部的 net_var 定义）
atlasname = onatlasname,
imagename = onimagename,
```

对外而言，**直接给属性赋值**：

```lua
-- ✅ 正确
inst.components.inventoryitem.atlasname = "images/inventoryimages.xml"
inst.components.inventoryitem.imagename = "wardrobe"

-- ❌ 错误：不存在这些方法
inst.components.inventoryitem:SetAtlas("...")
inst.components.inventoryitem:SetImage("...")
```

### `imagename` 不能带 `.tex` 后缀

`inventoryitem_replica:SetImage()`（line 132）自动追加 `.tex`：

```lua
function InventoryItem:SetImage(imagename)
    self.classified.image:set(imagename ~= nil and (imagename..".tex") or 0)
end
```

所以服务端赋 `"wardrobe.tex"` → 客户端变成 `"wardrobe.tex.tex"` → 图集中找不到 → `FROMNUM` 错误。

**正确**：`imagename = "wardrobe"`（不带后缀）。
